# Linux Server Management System (LinuxAI)

AI（OpenAI, Google Gemini, Kimi）の力を借りて、Linuxサーバの運用・管理を効率化するモダンなWebフロントエンド/システム管理ツールです。

対象となるLinuxサーバ内に直接Next.jsアプリケーションをデプロイすることで、ブラウザからセキュアかつ直感的にサーバの状態を確認し、自然言語から生成されたシェルコマンドを実行することができます。

## 🎯 主な機能

1. **直感的なダッシュボード (Server Portal)**
   - サーバのOS情報、稼働時間(Uptime)の表示
   - CPU、メモリ(RAM)、ディスク使用率のリアルタイム状況の表示
   - 美しいグラスモーフィズム(Glassmorphism)とダークモードを活用したUI

2. **AI コマンド生成 & 実行ターミナル (AI Terminal)**
   - **OpenAI (GPT-4 / GPT-3.5) / Google Gemini (Gemini 1.5 Pro) / Kimi (Moonshot AI)** に対応
   - 「Nginxのログを確認して」「空き容量を調べて」など、自然言語による指示からLinuxコマンドを自動生成
   - ブラウザのUI上からワンクリックで生成されたコマンドを安全に実行し、結果をリアルタイム表示

3. **セキュアな設定管理**
   - AIプロバイダーの切り替え機能と、各種APIキーの設定・保存機能

## ⚙️ テクノロジースタック

* **Framework:** Next.js (App Router), React
* **Styling:** Tailwind CSS, Framer Motion, Radix UI
* **State Management:** Zustand
* **AI Integration:** OpenAI API, Google GenAI SDK

## 🚀 インストール & 起動要件

本システムは管理対象とするLinuxサーバ上に配置して実行します。

### 前提条件
- Node.js (v20以上推奨)
- npm

### インストール手順

```bash
# プロジェクトをクローン
git clone https://github.com/yuanzhengliu/linuxmanage.git
cd linuxmanage

# 依存パッケージのインストール
npm install

# Next.jsアプリケーションのビルド
npm run build
```

### 永続化とバックグラウンド起動 (systemd 推奨)
PM2等を使用せず、Linuxの標準的な `systemd` を使用して自動起動させる方法を推奨します。（設定方法の詳細は別途ドキュメントを参照してください）

一時的に起動をテストする場合は、以下のコマンドを実行します。
```bash
npm run start
```
起動後、ポート3000 (`http://localhost:3000` または `http://<サーバIP>:3000`) にアクセスします。

## ⚠️ セキュリティに関する注意
本システムは、ブラウザからサーバ上で任意のシェルコマンドを実行できる権限を持ちます。本番環境で運用する際は、以下のセキュリティ対策を必ず実施してください。

1. Nginx等のリバースプロキシを用いた HTTPS (SSL) 通信の導入
2. IP制限（社内ネットワークやVPN経由のみアクセス許可）または Basic認証等の設定
3. アプリケーションを実行するLinuxユーザーへの最小権限（sudoパスワードなし実行等の制限）の付与
