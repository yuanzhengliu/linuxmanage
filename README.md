# Linux Server Management System (LinuxAI)

AI（OpenAI, Google Gemini, Kimi）の力を借りて、Linuxサーバの運用・管理を効率化するモダンなWebフロントエンド/システム管理ツールです。

対象となるLinuxサーバ内に直接Next.jsアプリケーションをデプロイすることで、ブラウザからセキュアかつ直感的にサーバの状態を確認し、自然言語から生成されたシェルコマンドを実行することができます。

## 🎯 主な機能

1. **直感的なダッシュボード (Server Portal)**
   - 全ての情報を稼働中のサーバからリアルタイムに取得
   - Linuxの実際のOSユーザー名とパスワードを使用したSSH認証ログイン
   - OS情報、稼働時間(Uptime)、CPU、メモリ(RAM)、ディスク使用率の表示
   - 美しいグラスモーフィズム(Glassmorphism)とダークモードを活用したUI

2. **AI コマンド生成 & 実行ターミナル (AI Terminal)**
   - **OpenAI (GPT-4 / GPT-3.5) / Google Gemini (Gemini 1.5 Pro) / Kimi (Moonshot AI)** に対応
   - 「Nginxのログを確認して」「空き容量を調べて」など、自然言語による指示からLinuxコマンドを自動生成
   - ブラウザのUI上からワンクリックで生成されたコマンドを安全に実行し、結果をリアルタイム表示

3. **セキュアな設定管理**
   - AIプロバイダーの切り替え機能と、各種APIキーの設定・保存機能

## ⚙️ テクノロジースタック

* **Framework:** Next.js (App Router), React
* **Styling:** Tailwind CSS, Framer Motion, Radix UI
* **State Management:** Zustand
* **AI Integration:** OpenAI API, Google GenAI SDK

## 🚀 インストール & 起動要件 (Installation)

本システムは管理対象となるLinuxサーバ上に配置して実行し、バックグラウンド稼働（デーモン化）させます。

### 前提条件
- 対象OS: Ubuntu 22.04 LTS または Debian 系Linux (RedHat系の場合はパッケージマネージャが異なります)
- 必要な権限: `sudo` 権限を持つユーザー
- Node.js / npm の準備

### 1. Node.js のインストール
```bash
# パッケージリストの更新
sudo apt update

# Node.js (v20.x) のセットアップスクリプトを取得
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -

# Node.js と npm をインストール
sudo apt install -y nodejs
```

### 2. アプリケーション配置とソースコードの取得
システムを配置するディレクトリを作成し、GitHubからソースをクローンします。
```bash
# 配置先のディレクトリを作成し、権限を付与
sudo mkdir -p /opt/linuxai
sudo chown -R $USER:$USER /opt/linuxai

cd /opt
git clone https://github.com/yuanzhengliu/linuxmanage.git linuxai
```

### 3. パッケージインストールとビルド
依存ファイルをインストールし、本番環境用のビルド処理を実施します。
```bash
cd /opt/linuxai
npm install
npm run build
```

### 4. systemdを用いたサービス化（自動・永続稼働設定）
Linuxの標準機能である `systemd` を使用して、アプリケーションをバックグラウンドでデーモンとして動かします。

`/etc/systemd/system/linuxai.service` を新規作成し、以下の内容を記載します。
（※ `User=` と `Group=` は実際にシステムを操作するLinuxユーザー名に変更してください）

```ini
[Unit]
Description=Linux AI Server Management System
After=network.target

[Service]
User=YOUR_USERNAME
Group=YOUR_USERNAME
WorkingDirectory=/opt/linuxai
ExecStart=/usr/bin/npm run start
Environment=PORT=3000
Environment=NODE_ENV=production
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

サービスの有効化と起動を行います。
```bash
sudo systemctl daemon-reload
sudo systemctl enable linuxai
sudo systemctl start linuxai
# 状態確認
sudo systemctl status linuxai
```

ブラウザから `http://<サーバIP>:3000` にアクセスし、ログイン画面が表示されるか確認してください。
**初回ログイン時は、システムのインストール先であるLinuxサーバの実OSユーザー名とパスワードを入力して認証します。**
（アクセスできない場合は `sudo ufw allow 3000/tcp` 等でポートを開放してください）

## ⚠️ セキュリティに関する重要なお願い
本システムは、対象サーバに対してブラウザ経由で「任意のシェルコマンドを実行できる権限」を有しています！
実運用においては必ず以下の対策を実施してください。

1. **アクセス制限:** ファイアウォール（iptables/ufw等）で社内・VPNからのみポート3000を許可する。
2. **通信の暗号化と多段認証:** リバースプロキシ(Nginx等)を前段に置き、HTTPS (SSL) 通信とBasic認証を必ず導入する。
3. **最小権限の原則:** LinuxAIを実行するユーザーはrootを避け、sudoコマンドの実行範囲を必要最低限に絞る。
